# 📘 02 - React Project Structure

## 🧠 What To Learn
- Vite gives you a clean folder structure out of the box
- There is only **one HTML file** in the whole project (`index.html`)
- All your React code lives inside the `src/` folder
- `public/` is for static assets (images, icons) that don't get processed
- `package.json` tracks your project name, scripts, and dependencies
- You never touch `node_modules/` — it's auto-generated by `npm install`

## 🗂️ File by File Breakdown

| File/Folder | Purpose |
|---|---|
| `index.html` | The only HTML file — has `<div id="root">` where React mounts |
| `src/main.jsx` | Entry point — imports App and renders it into root |
| `src/App.jsx` | Root component — your actual UI lives here |
| `src/index.css` | Global CSS applied to the whole app |
| `src/App.css` | CSS specific to App component |
| `src/assets/` | Images, SVGs used inside components |
| `public/` | Static files served directly (favicon etc.) |
| `package.json` | Project metadata, scripts, dependencies list |
| `vite.config.js` | Vite configuration (mostly leave it as is) |
| `node_modules/` | All installed packages — never touch, never push |
| `.gitignore` | Tells Git what NOT to push (node_modules is in here ✅) |



## 🔑 Key Concepts

| Concept | What it means |
|---|---|
| Entry point | Where the app starts — `main.jsx` |
| Root component | The top-level component — `App.jsx` |
| Static assets | Files that don't change — live in `public/` |
| `node_modules` | Never push this to GitHub — it's in `.gitignore` |

## 🔁 How the Flow Works
```
index.html
  └── loads main.jsx
        └── renders <App />
              └── your UI appears in the browser
```
## 🗂️ Project Structure

```
01-first-react-app/
├── public/
│   └── vite.svg
├── src/
│   ├── assets/
│   │   └── react.svg
│   ├── App.css
│   ├── App.jsx          ⭐ root component
│   ├── index.css        ⭐ global styles
│   └── main.jsx         ⭐ entry point
├── .gitignore
├── index.html           ⭐ the ONE html file
├── package.json         ⭐ project info + dependencies
├── package-lock.json
└── vite.config.js       ⭐ vite settings
```

> `index.html` loads `main.jsx` which renders `App.jsx` - that's the entire flow. Everything else is just support files.
